# 保险独立比价平台 — 设计方案

> 定位：只开方不卖药，做保险行业的"独立用药顾问"
> 对比免费 + 理赔协助付费 + B2B SaaS

---

## 一、核心定位

**不是保险导购，是独立保险工具平台。**

- 不做产品推荐，只做客观比对
- 不拿保险公司佣金，收入来自用户和B端
- 理赔协助作为核心盈利点

---

## 二、数据模型

### Product（产品）
```
id, category (enum: 重疾/医疗/意外/寿险/年金/惠民保/家财/车险), 
company, name, type (定期/终身/消费/返还),
coverage_items (JSON), exclusions (JSON), 
waiting_period, deductible, premium_formula (JSON),
claim_ratio, min_age, max_age, min_insured_term,
underwriters (JSON), status
```

### ProductDimension（产品维度评分）
```
id, product_id (FK), dimension_id (FK),
value, raw_text, score (0-100), evidence (JSON)
```

### Dimension（评分维度）
```
id, name (保障额度/覆盖病种/豁免条款/等待期/免赔额/赔付比例/保费/公司评级/理赔率),
category, weight, normalize_func
```

### UserProfile（用户画像）
```
id, age, gender, occupation, income_range,
health_status, smoke, has_insured, dependents,
existing_products[]
```

### Comparison（比对会话）
```
id, user_id (FK), created_at, report_snapshot (JSON)
```

### ClaimAssistant（理赔协助）
```
id, user_id (FK), product_id (FK),
phase (enum), required_docs (JSON),
timeline (JSON), status
```

## 三、MVP分期规划

### P0 — 核心验证
- 产品数据录入工具（初期手动录入50款）
- 维度标准化引擎
- 简易对比器（同类产品逐项对比）
- 需求测评问卷（5-8题出风险画像）

### P1 — 体验提升
- 条款结构化NLP（AI辅助抽取）
- 保费试算器
- 对比报告生成（PDF/分享卡片）
- 用户账户系统

### P2 — 商业闭环
- 理赔助手（流程+材料清单+时间线）
- B2B API（SaaS输出给独立代理人）
- 付费墙（高级报告/无限对比 订阅制）

## 四、评分引擎 API

```
POST /api/compare
{
  "category": "重疾险",
  "user_profile": { "age": 30, "gender": "M", "smoke": false },
  "product_ids": ["p1", "p2", "p3"],
  "weights": { "保费": 0.3, "保障范围": 0.3, "理赔率": 0.2, ... }
}
→
{
  "rankings": [
    { "product_id": "p1", "total_score": 87, 
      "dimensions": {...}, "reason": "..." }
  ],
  "best_match": "...",
  "disclaimers": ["..."],
  "version": "v1.0"
}
```

**核心原则：**
- 每次报告附带版本号和免责声明
- 评分算法版本化管理，可追溯
- 产品数据标注来源和时间戳
- 所有维度对用户透明

## 五、产品数据采集策略

### 首批覆盖50款（分险种）

| 险种 | 数量 | 选品逻辑 |
|------|------|----------|
| 百万医疗 | 8 | 好医保、尊享e生、平安e生保等 |
| 重疾险 | 10 | 超级玛丽、达尔文、i无忧、妈咪保贝等 |
| 意外险 | 6 | 按职业类别各2款 |
| 定期寿险 | 6 | 定海柱、大麦、擎天柱等 |
| 年金/增额寿 | 8 | 传统/分红/万能 |
| 惠民保 | 6 | 各城市代表产品 |
| 家财险 | 3 | 基础型/综合型 |
| 车险 | 3 | 按服务等级 |

### 数据来源（按优先级）
1. 保司官网条款PDF → 人工填入结构化表单
2. 银保监会备案信息 → 验证基础字段
3. 第三方聚合平台（深蓝保/慧择）→ 交叉核对
4. 代理人版计划书 → 保费数据
5. 经纪人询价系统 → 实际费率+核保规则

### 数据质量规则
- 双签制（录入人+复核人）
- 关键字段附原文截图/链接
- 保费数据标注采集日期
- 每周交叉验证（随机抽20%）
- 产品变更设置人工提醒

### 规模化路径
```
Phase 1 人工录入 → 50款，2人周
Phase 2 模板化复制 → 200款，建立SOP
Phase 3 半自动 → LLM辅助条款结构化（人工复核）
Phase 4 社区共建 → 开放经纪人提交入口
```

## 六、商业模式与财务模型

### 收入组合

| 层 | 产品 | 定价 | 作用 |
|----|------|------|------|
| 引流 | 对比+测评 | 免费 | 建立信任 |
| 转化 | 高级报告/保单体检/续保提醒 | ¥9.9-29/月 | 稳定现金流 |
| 盈利 | 理赔协助（按次） | ¥99-499/单 | 核心利润 |
| 延伸 | B2B SaaS经纪人账号 | ¥99-299/月 | 企业收入 |

### 12个月财务推演（单人运营）

**固定月成本：** ¥1,050-1,550（服务器¥300 + 域名¥50 + API¥200 + 推广¥500-1,000）

| 指标 | M1-M3 | M4-M6 | M7-M9 | M10-M12 |
|------|-------|-------|-------|---------|
| 累计注册 | 500 | 2,000 | 5,000 | 10,000 |
| 订阅用户 (3%) | — | 60 | 150 | 300 |
| 月理赔案件 | — | — | 6 | 19 |
| B2B账号 | — | 5 | 15 | 30 |

**M12月收入：** 订阅¥5,970 + 理赔¥5,681 + B2B¥2,970 = **¥14,621**

**盈亏平衡点：** 第5-6个月，前4个月净亏损约¥5,200

### 需持续追踪的3个核心指标
1. 注册到理赔转化率 — 理赔线天花板
2. 经纪人续约率 — B2B可持续性
3. 内容获客成本 — 能否规模化

## 七、市场竞品分析

### 国内主流（均收佣金）
| 平台 | 模式 | 比价 | 理赔 | 佣金 |
|------|------|------|------|------|
| 深蓝保 | 测评+导购 | ✓ | ✗ | ✓ |
| 蚂蚁保金选 | 评分+导购 | ✓ | 微保工具 | ✓ |
| 慧择 | 内容+导购 | ✓ | ✗ | ✓ |
| 多保鱼 | 评测+导购 | ✓ | ✗ | ✓ |

### 海外模式
| 平台 | 市场 | 模式 |
|------|------|------|
| PolicyBazaar | 印度 | 比价+佣金 |
| NerdWallet | 美国 | 综合金融对比+佣金 |
| Compare.com | 美国 | 比价引擎+佣金 |
| 10Life | 香港 | 精算评分+持牌经纪 |
| ACAP | 美国 | 纯理赔协助（无比价） |

**结论：** "比价+理赔+零佣金"的模型在全球范围内无成熟先例，是蓝海空白。

## 八、落地路径 — 先学再做

### 学习清单

| 技能 | 学习方式 | 时间 |
|------|---------|------|
| 保险条款阅读 | 下载3款在售条款PDF对照读 | 2周 |
| 对比维度识别 | 拆解深蓝保/慧择的测评文章 | 1周 |
| 理赔流程 | 知乎搜索各险种理赔流程 | 1周 |
| 真实案例 | 裁判文书网搜理赔纠纷判例 | 长期 |

### 最佳路径建议
先考代理人 → 入职经纪公司6-12个月 → 积累产品和理赔经验 → 再出来做独立平台

### 用AI辅助工作流
- **你**：提问题、定方向、复核结果、做判断
- **AI**：搜索整理、结构化输出、填模板、出初稿
- **阶段产出**：2周内可产出产品对比表 + 3篇内容 + 理赔手册

## 九、技术架构建议

```
前端: React/Vue + Tailwind, 移动端优先, 对比视图 grid 布局
后端: Node.js (Express/Nest) 或 Python (FastAPI)
数据: PostgreSQL (JSONB), Redis (缓存热门对比)
NLP: 初期人工, 后期 LLM + PaddleNLP
评分: 权重配置化, 用户可调维度权重, 引擎纯函数无状态
```

### 手动版MVP
不做网站/app，用微信群/公众号接需求 → 手动出对比报告（Excel/Notion/PDF）

**先手动验证三个问题：**
1. 有没有人愿为中立对比找你
2. 出险后愿不愿回来找你帮忙理赔
3. 愿不愿意付费

## 十、关键原则

- 所有对比报告附带版本号和"仅供参考，不构成投保建议"声明
- 评分算法版本化管理，可追溯
- 产品数据维护日志，记录来源和时间戳
- 所有比较维度对用户透明，可查原始条款
- 先人工、后自动；先验证、再投入
